### 技术选型

| 层 | 技术 | 理由 |
|---|------|------|
| 3D 图谱 | Three.js + React Three Fiber | 60fps 粒子图，支持缩放/旋转 |
| 数据图表 | Recharts | 轻量，React 原生 |
| 时间线 | visx | Airbnb 出品，灵活 |
| 实时通信 | WebSocket (FastAPI 原生) | 零额外依赖 |
| 分享卡片 | html2canvas | 前端截图，即时生成 PNG |
| 粒子背景 | tsParticles | 轻量，配置丰富 |

## 四、两套方案对比

| 维度 | A: Neural Matrix | B: Crystal Core |
|------|:--:|:--:|
| **视觉冲击** | ★★★★★ 赛博朋克，一眼惊艳 | ★★★★ 精致专业 |
| **信息密度** | 高（适合大屏） | 中（适合笔记本） |
| **学习曲线** | 陡（需要理解 3D 导航） | 平（传统仪表板布局） |
| **社交传播** | 酷炫截图易传播 | 数据报告适合 LinkedIn |
| **移动端** | 差（3D 性能） | 好（卡片布局响应式） |
| **开发成本** | 高（Three.js 3D 调试成本） | 中（D3 + Recharts 成熟） |
| **扩展性** | 3D 粒子图加新节点简单 | 仪表板加新卡片简单 |
| **主题化** | 暗色固定 | 明/暗双主题 |

## 五、推荐策略

### 分阶段实施

```
Phase 1 (2 周): Crystal Core 基础版
  - 4 个 KPI 卡片 (KG/PM/QL/Health)
  - 2D 力导向知识图谱 (D3)
  - 情景记忆时间线 (visx)
  - 明/暗主题切换

Phase 2 (1 周): Crystal Core 完整版
  - Q 表热力图
  - Dream Cycle 环形进度
  - Agent Hub
  - 分享卡片生成

Phase 3 (2 周): Neural Matrix 3D 版
  - Three.js 3D 知识图谱
  - 粒子背景 + 霓虹效果
  - 社交分享模板
  - 导出功能
```

### 最终建议

**先做 Crystal Core（方案 B），后做 Neural Matrix（方案 A）。**

理由：
1. Crystal Core 开发快（D3+Recharts 成熟），2 周可上线
2. Crystal Core 信息架构好，先验证"用户需要看什么"
3. Neural Matrix 在验证后做，避免 3D 可视化"好看但没用"
4. 两套方案共享同一个 FastAPI 后端，前端切换主题即可
